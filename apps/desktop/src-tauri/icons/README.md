`tauri.conf.json` references `icons/icon.png` for the app/bundle/tray icon. `icon.png` here is a minimal 512×512 placeholder (three ascending bars, generated programmatically) — enough that `cargo build`/CI don't fail on a missing file, but explicitly **not** real branding (see `../../../docs/ux/`). Before running `tauri build` to actually produce a distributable installer, replace it with real artwork and run `tauri icon` to generate the platform-specific variants (`.ico`, `.icns`, sized PNGs) that bundling needs.
